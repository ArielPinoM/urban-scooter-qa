# US-71: DELETE /api/v1/courier/:id documentation incorrectly indicates that id is sent in the body instead of being a path parameter

# Key Details

## Severity
🟠 Major

## Priority
🟧 High

## Environment

- API Documentation (apidoc)

- Postman 12.16.4

## Component
API - Delete Courier

## Description

The specification of the `DELETE /api/v1/courier/:id` endpoint in the documentation generated with apidoc shows that the `id` parameter should be sent in the request body (JSON):

```json
{
  "id": "3"
}
```

However, the endpoint is correctly implemented as a path parameter (`/api/v1/courier/:id`), where the `id` should be in the URL (e.g., `/api/v1/courier/3`). If the documentation is followed literally and the id is sent only in the body, the request fails because the route does not include the identifier.

The documentation must accurately reflect the actual API contract.

### Preconditions

- Server is running.

- A courier with known `id` exists.

### Steps to Reproduce

1. Follow the documentation: send `DELETE /api/v1/courier/` (without id in the URL) with body { "id": "<courier_id>" }.

2. Observe the result.

3. Compare with the actual correct behavior: send `DELETE /api/v1/courier/<courier_id>` (without body or with empty body).

### Expected Result (per documentation)

- The documentation should specify that `id` is a route parameter (part of the URL) and not a body field.
