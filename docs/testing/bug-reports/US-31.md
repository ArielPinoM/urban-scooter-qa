# US-31: Push notification not received 2 hours before delivery deadline

# Key details

## Severity
🟠 Major

## Priority
🟧 High

## Environment
- Android Emulator: Galaxy A3, Android 9
- Postman 12.17.3

## Component
Mobile - Notification

## Description

### Preconditions

- Server time is before 21:59.
1. Create a courier account with "login": "apm96", "password": "1234", "firstName": "Ariel" using POST /api/v1/courier.
2. Create an order using POST /api/v1/orders with "deliveryDate": "<today's_date>" in the request body.
3. The mobile application has notification permissions granted.
4. Enter the backend URL in the login screen of the mobile application.
5. Sign in with the credentials of the courier account that was created.

### Steps to reproduce
1. Accept the order in the orders screen, "Todos los pedidos" tab.
2. Wait until server time reaches 21:59.
3. Monitor the emulator for the notification.

### Expected result
A push notification is received with the text: "2 horas hasta el final del pedido".

### Actual result
No notification is received.
