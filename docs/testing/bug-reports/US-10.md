# US-10: The canceled order remains visible to the user and its "cancelled" field is not updated

# Key details

## Severity
🟠 Major

## Priority
🟧 High

## Environment
- Opera 132, 1280x720 (Chrome blocked by [US-1](./US-1.md))
- Postman 12.16.4
- Api Ez-scooter version 1.0.0
- Back-end
    - Node.js v12.17.0
- PostgreSQL database

## Component
Order Status

## Description
When the order is canceled, it is not removed or hidden. When checking its number again in the "Estado del pedido" screen, the order information appears again.

When verifying the database, the `cancelled` field in the `Orders` table remains `false`.

### Preconditions
- There is a valid order that has not been accepted and whose tracking number is known.

### Steps to reproduce
1. In Opera, place a valid order and note its tracking number.
2. Click "Estado del pedido", enter the number and click "¡Vamos!".
3. Click "Cancelar el pedido".
4. In the popup, confirm the cancellation.
5. In the popup, click "Bien".
6. Repeat step 2.

### Expected result
After cancellation, the order disappears from the system. When attempting to look it up, an error message "No existe tal pedido" is shown.

### Actual result
- The canceled order reappears with all its data.
- In the database, the `cancelled` field remains `false` for that record.
