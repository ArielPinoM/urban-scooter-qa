# US-70: DELETE /api/v1/courier/:id deletes the courier but does not remove linked orders in the Orders table

# Key Details

## Severity
🟠 Major

## Priority
🟧 High

## Environment
- Ez-scooter API v1.0.0
- Postman 12.17.3

## Component
API - Delete Courier

## Description

According to back-end requirements, when deleting a courier account, linked orders in the `Orders` table should be deleted.

However, when sending a `DELETE /api/v1/courier/:id` request with a valid courier ID that has accepted orders, the courier is correctly removed from the `Couriers` table, but all associated order records (field `courierId`) remain intact in the `Orders` table.

### Preconditions

A courier exists in the `Couriers` table with a known `id`.

### Steps to Reproduce

1. In Postman, create a `POST` request to `/api/v1/orders` with the body:

```json
{
    "firstName": "Ariel",
    "lastName": "Pino",
    "address": "1st Street",
    "metroStation": 2,
    "phone": "+1 999 999 99 99",
    "rentTime": 1,
    "deliveryDate": "2026-07-05",
    "comment": "Test comment",
    "color": [
        "BLACK"
    ]
}
```
2. Send the request.

3. Copy the `track` returned in the response.

4. Create a `GET` request to `/api/v1/orders/track?t=<created_order_track>`.

5. Send the request.

6. Copy the `id` of the order returned in the response.

7. Create a `PUT` request to `/api/v1/orders/accept/:<created_order_id>?courierId=<courier_id>`.

8. Send the request.

9. Create a `DELETE` request to `/api/v1/courier/<courier_id>`.

10. Execute `select * from "Couriers" where "id" = <deleted_courier_id>;`.

11. Observe that the record for that `id` no longer exists.

12. Execute `select * from "Orders" where "courierId" = <deleted_courier_id>;`

### Expected Result

- The courier is deleted from `Couriers`.

- All orders in `Orders` whose `courierId` matched the deleted `id` have been removed.

### Actual Result

- The courier is correctly deleted from `Couriers`.

- Orders remain in `Orders` with the `courierId` intact, creating orphaned records.
